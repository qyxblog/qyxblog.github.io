诚信

引入

诚信是立身之本。一个人如果失去了诚信，将在社会上没有立足之地。诚信是中华民族的传统美德，是社会主义核心价值观的重要内容，是做人的根本。古往今来，人们写出了很多关于诚信的名言和故事，由此可见诚信的重要性。

那么，何为诚信？

诚信是由两个字组成的：“诚”与“信”。“诚”即真实，实在。“信”则不仅要求人们说话诚实可靠，切忌大话、空话、假话，而且要求做事也要诚实可靠。而“信”的基本内涵也是信守诺言、言行一致、诚实不欺。简单说诚信就是一个道德范畴，是公民的第二个“身份证”，是日常行为的诚实和正式交流的信用的统称。

诚信的故事

那么，我给大家分享一下诚信的小故事。

曾子杀猪

曾子是孔子的学生，他博学多才，且十分注重修身养性，德行高尚。一次，他的妻子要到集市上办事，年幼的孩子吵着要去。妻子不愿带孩子去，便对他说：“等妈妈回来将家里的猪杀了给你吃。”孩子听了，非常高兴，不再吵着要去集市了。这话本是哄孩子说着玩的，过后，妻子便忘了。不料，曾子却真要杀猪。妻子说，“我是骗他的啊，你怎么当真呢！”曾参说：“孩子是不能欺骗的。孩子小，只得学习别人的样子，尤其是以父母作为生活的榜样。今天你欺骗了孩子，明天孩子就会欺骗你、欺骗别人。“

商鞅的立木为信

春秋战国时，商鞅在主持变法。当时处于人心惶惶之际，为了树立威信，推进改革，商鞅下令在都城南门外立一根三丈长的木头，并当众许下诺言：谁能把这根木头搬到北门，赏金十两。围观的人不相信如此轻而易举的事能得到如此高的赏赐，结果没人肯出手一试。于是，商鞅将赏金提高到50金。重赏之下必有勇夫，终于有人站起将木头扛到了北门。商鞅立即赏了他五十金。商鞅这一举动，在百姓心中树立起了威信，而商鞅接下来的变法就很快在秦国推广开了。新法使秦国渐渐强盛，最终统一了中国。

小结

由此，我们可以看出诚信的重要性，那么如果社会没有诚信呢？

社会失掉了诚信

社会必将失去基本的公平正义，而且人与人之间也就不会有基本的信任，失去了信任人本身存在也就失去了道德的支撑，没有了道德我们也就失去了生存的基本的进化之根！那必将是人类的灾难。

守住身边的诚信

所以我们更要诚实守信，答应别人的事说到做到，不欺骗他人！

Win端完美深色2，带你打出趋于完美的组合拳！

🖊前言

我之前写过一篇让Windows自动切换深色和浅色模式的文章，在那篇文章里，我们使用的是计划任务，但是它有一个缺点：比如说我把自动切换深色的时间设在18:00，那么如果我在18:00没开电脑，而在18:00后开了电脑，那么就不会自动切换深色模式，浅色模式的切换，缺点也是如此。

那么有没有更好的方案？

🤜第一式：Windows-Auto-Night-Mode

Windows-Auto-Night-Mode是一个在Github上开源的软件，意味着纯净、免费、不作恶。

安装界面多语言可选，但是没有中文，不过不用担心，安装完毕后，主界面是中文的。

你可以通过自定义时间或者日出至日落来自动切换深色和浅色。

点击应用程序，可以进一步自定义

最棒的是，它支持切换壁纸！我可以在白天使用浅色壁纸，在夜晚自动切换为深色壁纸！

⚔第二式：Twinkle Tray

如果你是笔记本用户，你可以按动笔记本上的亮度切换，以此调整亮度

但如果你是台式机用户…是的，我知道你不想打开显示屏复杂的菜单去调亮度，所以就有了Twinkle Tray

和Windows-Auto-Night-Mode一样，它也在Github上开源。

首次使用，它会躲在你的托盘里。当你把鼠标移到它的图标上，再滚动滑轮时，电脑屏幕的亮度也会跟着改变。

点击托盘图标也可以调亮度。

它还支持根据时间自动切换显示屏的亮度

当然也少不了快捷键控制亮度，笔记本般的体验！

值得一提的是，Twinkle Tray对多显示器有着较好的支持。

📕总结

有了这两款应用，你的电脑基本就可以实现完美的Dark Mode了！

未加密的Git将不受支持

🤒不好的消息

刚刚把Github搭博客的文章写完，这边就出事了：

Github:未加密的Git协议即将成为历史

摘自IT之家

Github发表博客，表示：将提高Github上的Git协议的安全性，将删除未加密的Git协议。

GitHub 官方宣布，正在更改 SSH 支持的密钥并删除未加密的 Git 协议，具体来说：

  • 取消对所有 DSA 密钥的支持
  • 添加对新添加的 RSA 密钥的要求
  • 删除一些旧的 SSH 算法（HMAC-SHA-1 和 CBC 密码）
  • 为 SSH 添加 ECDSA 和 Ed25519 主机密钥
  • 关闭未加密的 Git 协议

GitHub 表示，这一改变仅影响使用 SSH 或 git:// 进行连接的仓库，以 https:// 开头的连接协议完全不受影响。

GitHub 官方认为，公钥密码术依赖于安全算法和足够强大的密钥来保持安全。“更少的位数”通常意味着“更容易暴力破解”，而较旧的算法也已经出现了攻击方式。考虑到计算能力的变化、新的攻击等，在 2001 年被认为是安全的密钥，今天可能不再安全了。

IT之家了解到，GitHub 在上个月还宣布取消对 HTTPS 密码的支持，总之就是要确保 GitHub 用户数据的安全。官方表示，将继续观察使用 SHA-1 算法的 RSA 密钥，一旦出现风险或者过时，将向社区通报弃用它。

😀不过

不过我教你们的方法恰好不是SSH

📕总结

请大家放心！Github对SSH的封禁完全不影响我们继续建网站！
